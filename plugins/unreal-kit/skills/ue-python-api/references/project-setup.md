# Project Setup

## Quick Start

Run once per machine from a terminal:

```bash
<skill-dir>/bin/setup.cmd
```

Or directly with Python:

```bash
python <skill-dir>/bin/setup.py
```

## What Setup Does

The setup script auto-discovers everything from its position in the project tree — no user input required.

1. **Finds your `.uproject`** — walks up from the skill directory
2. **Finds the Engine directory** — walks up from `.uproject` looking for `Engine/Binaries/Win64/UnrealEditor-Cmd.exe`
3. **Writes `config.yaml`** — to `~/.claude/plugins/data/unreal-kit/config.yaml` (used by the runner for paths)
4. **Enables `bRemoteExecution`** — in `Config/UserEngine.ini` (per-user, not checked in). Allows running scripts from terminal via UDP
5. **Enables `bIsDeveloperMode`** — in `Config/UserEngine.ini`. Enables UE to generate Python API stubs
6. **Downloads API stubs** — from PyPI (`unreal-stub` package). Used by the AI to search the UE Python API
7. **Writes `setup-status.yaml`** — success marker in the skill directory

## After Setup

**Restart UE Editor** — the `bRemoteExecution` and `bIsDeveloperMode` settings only take effect on editor startup.

### Optional: Refresh Stubs

After restarting the editor (with Developer Mode now enabled), you can get project-specific stubs that include your project's custom types:

```bash
<skill-dir>/bin/setup.cmd --refresh-stubs
```

This copies stubs from `<Project>/Intermediate/PythonStub/unreal.py` (generated by UE when Developer Mode is on). These are more complete than the generic PyPI stubs.

## How to Tell If Setup Is Needed

**Default assumption: setup is complete.** Only investigate if you encounter:

- `setup-status.yaml` is missing from the skill directory
- Config validation errors from `ue_runner.py` (e.g., "uproject path not configured")
- `stubs/unreal.py` not found when searching the API
- Remote execution fails with "Editor not responding" and settings haven't been configured

To check: read `<skill-dir>/setup-status.yaml`. If it exists with `status: complete`, setup has been done.

## Troubleshooting

### Python Not Found

If `setup.cmd` fails because Python isn't on PATH:

1. **Use UE's bundled Python**: The engine includes Python at `Engine/Binaries/ThirdParty/Python3/Win64/python.exe`. Run setup directly:
   ```bash
   "<engine-path>/Binaries/ThirdParty/Python3/Win64/python.exe" "<skill-dir>/bin/setup.py"
   ```

2. **Use the Windows Python launcher** (`py`): Installed by default with Python from python.org

3. **Add Python to PATH**: Add your Python installation directory to the system PATH environment variable

### Stubs Download Failed

If PyPI is unreachable (firewall, no internet):

- Stubs are optional — scripts will still run, the AI just can't search the API
- After getting editor running with Developer Mode, use `--refresh-stubs` to copy from the project instead

### `.uproject` Not Found

The script walks up from `<skill-dir>` looking for `.uproject` files. If your skill directory isn't inside the project tree, setup can't auto-discover. In that case, use the interactive setup:

```bash
<skill-dir>/bin/ue-runner.cmd --setup
```
