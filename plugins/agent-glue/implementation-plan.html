agent-glue v1 Implementation Roadmap

Three parallel tracks. One v1 cross-subsystem gate: work-system w3 (cache substrate) -> graph-system g4 (submit() integration).

The one v1 dependency edge

  • • Within v1 there is exactly one cross-subsystem gate
  • • work-system w3 (Show-your-work-as-cache) gates graph-system g4 (submit() integration)
  • • Until w3 lands, graph nodes that delegate via submit() raise
  • • The first three graph-system increments (g1..g3) don't touch the work subsystem

Tracks develop in parallel

  • • Core lands first and unblocks everyone
  • • After core, the three tracks run independently until the gate
  • • Each subsystem's increments proceed in document order
  • • No backporting: changes that cross the boundary apply in one CL

Out of frame: post-v1

  • • claude-work-queue (4 increments) ships as docs in v1, code post-v1
  • • work-system claude_inference + claude_agent workers ship in the same post-v1 wave
  • • Example pipelines, full HTML render, async / streaming all post-v1
  • • v1 = 1 core + 7 work + 8 graph = 16 increments shipping together
