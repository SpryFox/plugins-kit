agent-glue Graph Runtime: Variant Dispatch + Fan-out

Per-node: validate input -> execute -> validate output -> write canonical -> dispatch by variant. Fan-out only on list[T] + parallel:true edges.

One pass per node

  • • Validate input against Topology.in
  • • execute() returns (Output, StateDelta)
  • • Validate output against Topology.out (discriminated union)
  • • Write canonical artifact if Outputs component present
  • • Dispatch via core.dispatch.dispatch on the variant's kind

Three Disposition branches

  • • Accepted: proceed normally to the matching edge
  • • AcceptedWithAudit: same downstream + stamp audit metadata
  • • Rejected: route to a Rejected-targeted edge if one exists
  • • No matching edge = clean terminal (the variant IS the recorded disposition)

Fan-out is opt-in

  • • Only fires when out = list[T] AND edge.parallel = true
  • • Per-node max_workers via ParallelSpec; no global concurrency limit in v1
  • • Accumulated PipelineState fields get a lock automatically
  • • Mixed-disposition lists route per element
