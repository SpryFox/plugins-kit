# Architecture

How the UE Python API skill works under the hood.

## UE Plugin Requirements

- **Required**: "Python Editor Script Plugin" (`PythonScriptPlugin`) — provides the `import unreal` module and the `-run=pythonscript` commandlet runtime. Without this plugin enabled in the `.uproject`, UE Python scripts cannot run.
- **Recommended**: "Editor Scripting Utilities" (`EditorScriptingUtilities`) — provides convenience classes like `EditorAssetLibrary`, `EditorUtilityLibrary`, and `EditorLevelLibrary`. Not required for `import unreal` to work, but needed for many of the patterns in the skill.

## Script Execution Modes

The terminal runner (`ue_runner.py`) supports two execution modes and auto-detects which to use:

| Editor open? | Method | How it works |
|--------------|--------|-------------|
| Yes | Remote execution | Sends script to the running Editor via UDP multicast (`upyrc` library). The Editor's `PythonScriptPlugin` executes the script in-process. |
| No | Headless commandlet | Launches `UnrealEditor-Cmd.exe -run=pythonscript -script=<path>`. Starts a headless Editor instance, runs the script, and exits. |

Remote execution is slightly faster since the Editor is already loaded. The commandlet must load the engine from scratch.

### Manual execution methods

These bypass the terminal runner:

```
# Output Log (switch dropdown to Python) — requires Editor open
py "C:/path/to/script.py"

# Startup scripts (auto-run on editor load)
# Place .py files in <Project>/Content/Python/

# Commandline (headless) — no Editor needed
UnrealEditor-Cmd.exe project.uproject -ExecutePythonScript="C:/path/to/script.py"
```

## UE-side Dependency Management

Scripts running inside UE use a separate Python environment from the host. Third-party packages are installed into UE's embedded Python via **unreal-pip**.

The `ensure_dependencies()` function in `lib/bootstrap.py` reads `requirements.yaml`, checks what's installed, and uses unreal-pip for anything missing. For the bootstrapping flow, see `references/script-bootstrap.md`. For the unreal-pip API, see `references/unreal-pip.md`.

## API Stubs

The `stubs/unreal.py` file contains type stubs for the UE Python API. Two sources:

- **PyPI stubs** (`unreal-stub` package) — generic, covers the standard UE API. Downloaded automatically by bootstrap.
- **Project-specific stubs** (`<Project>/Intermediate/PythonStub/unreal.py`) — includes project-specific custom types. Generated by UE when Developer Mode is enabled. Bootstrap copies these when available, overriding the PyPI stubs.
