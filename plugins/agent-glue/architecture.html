agent-glue Subsystem Dependency Graph

Four-subsystem DAG. v1 ships three active tracks; claude-work-queue and the work-system claude-worker increment are deferred to post-v1.

The DAG is strict

  • • core has no upstream dependencies
  • • everyone imports from core
  • • only work-system's claude workers import claude-work-queue
  • • only graph-system imports work-system (via submit())
  • • no cycles, ever

Cross-subsystem interfaces

  • • graph -> work: agent_glue_lib.work.submit()
  • • work claude-workers -> queue: submit() / wait()
  • • * -> core: loader, components, Disposition
  • • submit() is the entire surface graph sees of work

What the deferred treatment means

  • • claude-work-queue: design + docs ship in v1, no Python
  • • claude_inference + claude_agent workers: depend on the queue, ship in the same post-v1 wave
  • • openrouter covers v1 LLM needs; deferral unblocks nothing
  • • the wire format and consumer API are already locked
