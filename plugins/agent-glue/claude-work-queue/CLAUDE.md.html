# claude-work-queue subsystem context The **claude-work-queue** subsystem of agent-glue. A standalone primitive for handing Claude a unit of work and getting the result back. The capability has three parts: a **queue** that holds work items, a **signal** that tells Claude work is available, and an **execute-and-report** loop where Claude does the work and writes the result back to a location the requester can find. This subsystem is consumable directly (any caller wants Claude to do something + come back with a structured result) and is also the substrate the work subsystem's `claude_inference` and `claude_agent` workers will dispatch through. ## Internal lib layout ``` agent_glue_lib/claude_work_queue/ __init__.py # exposes submit(), wait(), result(), typed errors storage.py # read/write WorkItem and WorkItemResult yamls against / claim.py # atomic claim via os.rename (pending/ -> in_progress/) report.py # result write + schema validation + done/ promotion ``` ## Where to find things | Topic | Document | |---|---| | The queue + signal + execute-and-report primitive; the three locked design decisions (file-based storage, Stop-hook signaling, open-to-any-writer scope); WorkItem and WorkItemResult wire format | DESIGN.md.html | | Entities + components (WorkItem, WorkItemResult; authored in the first implementation increment) | ARCHITECTURE.md.html | | Build increments and acceptance criteria | IMPLEMENTATION-PLAN.md.html | | Entity-type definitions and component schemas | entities/ and components/ |
